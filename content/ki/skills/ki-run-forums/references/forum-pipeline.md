# Forum pipeline

`draft → validation → receipt → open → discussion/evidence → ballot instructions → close → tally → resolved → Policy/Commands authorized → execution/reconciliation`

## Proposal minimum

Title, purpose, exact command payloads, proposer, Forum, Organization, eligibility rule, open/close time, passage rule, dependencies, conflicts, costs, affected Policies/resources, feature gates, receipts, and revision lineage.

## Separation

- Ballot: equal governance voice.
- Position: optional information signal under a separately activated policy.
- Resolution: deterministic application of Forum rules.
- Policy: authoritative enacted state.
- External operation: separately tracked consequence.

Never infer a Member's ballot from history. Ki may summarize likely considerations but must request explicit instruction where required.
