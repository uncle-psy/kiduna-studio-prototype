# Run Forums

Read [references/forum-pipeline.md](references/forum-pipeline.md).

## Procedure

1. Resolve exact Organization, Forum, eligible Membership, proposer authority, and Forum policy.
2. Draft a versioned proposal containing exact Commands and parameters, dependencies, conflicts, costs, feature gates, and machine receipts.
3. Validate opening requirements and create the proposal Action.
4. Keep discussion evidence, ballot instruction, and any optional decision-market position separate.
5. Let each eligible Member cast one equal ballot through confirmed Envoy instruction.
6. Apply recusals and eligibility at the deterministic boundary.
7. Close at the policy-defined deadline, resolve under the exact rule, and create a Record.
8. If passed, enact versioned Policy or authorized Commands; do not equate passage with external settlement.

## Language and boundaries

Say Forums, not markets, in member-facing copy. Deciding is never fundraising. Optional positions are information only and never replace one-Member/one-ballot governance.

In prototype mode, simulated ballots and positions stay in demo partitions and cannot move real funds or create real governance authority.
